##总结
字节流与字符流不同在于前者按字节读写，而后者按字符读写，例如分别用FileInPutStream和FileReader对java文件和txt文本文件读取，然后打印输出到控制台，结果发现英文和阿拉伯数字都可以正常显示，而对于中文，只有FileReader读取java文件输出结果能正常显示，其余均为乱码。
乱码原因有两种：
第一种是中文字符在Java里使用UTF-16对其进行编码转换为二进制进行存储，即每个中文字符需要占据两个字节的存储空间，而字节流读取时每次只能读取某个字符的其中一个字节。
第二种是编码格式的问题，在用FileReader对txt文件进行读取时，输出中文仍显示为乱码，这是因为在Windows操作系统下，txt文件默认的编码格式是ANSI，java文件默认编码是UTF-8，而Java中字符的默认编码格式是UTF-16，因此java文件的中文字符可正常显示，txt文件中文则为乱码。
